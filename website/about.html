About

Not a framework. Not a runtime. A workflow pack.

Open Tools is a collection of markdown files that opencode reads. No dependencies. No build step. No lock-in.

Why Open Tools Exists

General-purpose AI assistants don't enforce git safety. They don't prevent architecture drift. They don't maintain persistent kanban boards. They don't remember what happened in the last session.

Open Tools fixes this. It gives you specialized agents with explicit permissions, bounded domains, and memory across sessions. Each agent does one thing well. Together they cover the entire software lifecycle.

What It Is

Open Tools is a workflow pack for opencode — the AI coding tool. It contains:

  • 27 agents — Specialized AI personas with rules, permissions, and a purpose
  • 30 commands — Slash commands that invoke agents with specific prompts
  • 13 skills — Reusable instruction sets any agent can load
  • 6 packs — Curated groups of commands for specific workflows
  • 4 templates — Kanban, ADRs, repo maps, and session logs

What It Is Not

Open Tools is not a runtime. It doesn't execute code. It doesn't have a server. It doesn't store data. It's markdown files that opencode reads and follows.

You copy the agents into your project. Restart opencode. Run /toolbox. That's it.

The Safety Model

Every agent follows strict safety rules:

  • Review and audit agents are read-only by default
  • Memory agents write only under .opencode/
  • No agent stores secrets, tokens, or credentials
  • Destructive git commands are forbidden unless confirmed
  • All agents follow safe-git-workflow — no force-push, no git reset --hard

Values
What guides the project.

Bounded Domains

Every agent owns one domain. No overlap. No bleed. When a task falls outside an agent's boundary, it hands off to the right agent.

Explicit Contracts

Input schemas, output schemas, validation gates. Every agent validates its input before executing. Every agent logs its output.

Zero Dependencies

No runtime. No server. No build step. Markdown files that opencode reads. Copy them in. Start using them. That's the entire installation.

Memory First

Context survives token limits. Kanban boards, session logs, ADRs, and context directories preserve your work state across sessions.

Safety by Default

Read-only agents can't edit files. Memory agents write only to safe directories. Secrets are never stored. Destructive commands require confirmation.

Composable

Agents, commands, skills, and packs compose freely. Load a skill into any agent. Run any pack. Mix and match for your workflow.
