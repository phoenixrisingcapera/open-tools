How It Works
Four primitives. One system.
Open Tools is built on agents, commands, skills, and packs. Each piece does one thing well. Together they cover the entire software lifecycle.
⬡

Agents

Specialized AI personas with rules, permissions, and a purpose. Each agent owns one domain — code review, security, deployment, or design.

▶

Commands

Slash commands that invoke agents with specific prompts. /review-diff triggers the PR reviewer. /security-check triggers the sentinel.

◈

Skills

Reusable instruction sets any agent can load. Git safety, architecture drift prevention, data presentation — enforced across your workflow.

▣

Packs

Curated groups of commands for specific workflows. The preflight pack runs before you push. The memory pack preserves context.

⬢

Advanced Tools

Citation management, plugin validation, MCP server setup, skill building, and structured data presentation for complex workflows.

◎

Safety Model

Review agents are read-only. Memory agents write only under .opencode/. Secrets are never stored. Destructive git commands are forbidden.
