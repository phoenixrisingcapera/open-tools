# Public Repo Polish

Use this skill to make a public repo easier to trust, install, and share.

## Checklist

- Clear one-sentence purpose at the top of `README.md`.
- Quick install instructions.
- Copy-paste examples.
- What is included and what each item does.
- Safety/trust section for permissions, secrets, and destructive actions.
- License file.
- Contribution or extension guidance.
- Screenshots or output examples when relevant.
- Changelog or release notes once the repo has users.

## README Shape

1. What this is.
2. Why use it.
3. Install.
4. Commands/examples.
5. Included assets.
6. Safety model.
7. How to customize.
8. License.

## Public Trust Rules

- Do not include private workspace paths in reusable docs except as examples clearly marked local-only.
- Do not include secrets or internal deployment values.
- State permissions plainly.
- Prefer small, composable assets over a giant opaque prompt.
